Linux Python Installation and Configuration

Options

If you are running a more recent Linux distribution, you probably already have Python-2.7. In this case, using pip to install the needed modules is the best way to go. Otherwise, you'll have to build Python-2.7 from source.

Alternatively, I have provided a pre-built environment (here) compiled on a Red Hat EL Workstation 6.6 system. It will extract a directory named python27-linux than contains all the modules and shared objects required to run VeraView. It also includes a file named setenv.sh in the top directory that can be sourced from a Bash shell to set all the needed environment. You can extract it with a simple tar command: 

$ tar xvfj python27-linux.tar.bz2

Install with Pip

If you don't have Pip, grab the source from here and run this with your Python-2.7 executable: 

$ python get-pip.py

This should install a script named pip in your Python directory. It can be used to install (download source and build) any modules you're missing except for wxPython. It will take care of downloading dependent modules as well. You can test to see if you need a module by trying to import it in a Python script or interactively. The modules you need are:

  • h5py
  • matplotlib
  • numpy (installed by h5py)
  • Pillow (PIL)
  • pyparsing (installed by matplotlib)
  • python-dateutil (installed by matplotlib)
  • six (installed by matplotlib)

You install a module with a simple command, such as: 

$ pip install h5py

Build wxPython

Unfortunately, you'll have to build wxPython from source, available here. You must run the wxPython/build-wxpython.py script with your Python-2.7 executable, and there are a couple of arguments you should set. For example, if your Python-2.7 was installed in /usr/local, you might do the following: 

$ python build-wxpython.py --install \
--prefix=/usr/local \
--wxpy_installdir=/usr/local/lib/python2.7/site-packages
