Windows Python Installation and Configuration

Options

I've successfully run the VeraView python application under Anaconda and ActiveState The Anaconda path is faster and does not require elevated privileges-- you can do a single-user install into any directory for which you have permissions.

Note I will zip up an installed Anaconda environment and provide a setup batch file to set the correct paths. This could be the basis for a VeraView all-in-one Windows install.

Anaconda

You can get the Anaconda Python-2.7 installer from here. Install it anywhere you have permissions. It provides all the required modules except wxPython, including:

  • h5py
  • matplotlib
  • numpy
  • Pillow (PIL)

You will need to install wxPython, available from here. It's also on this site. If you did a per-user Anaconda install, you will get warning about not finding the Python path in the registry. Ignore that and enter the path to the Anaconda site-packages directory. For example, if you installed Anaconda to C:\Anaconda directory, the property directory is C:\Anaconda\Lib\site-packages.

You can run VeraView by specifying the path to the Python executable. For example, assuming you installed Anaconda to C:\Anaconda, the following command should work if the current directory is the VeraView directory: 

> c:\Anaconda\python veraview.py

ActiveState

If you have already installed ActiveState Python-2.7, or you just prefer it, you will need to make sure all the required modules are present. Installers for ActiveState itself are available from: here.

Copies of the installers for the required modules are on this site and can be installed one-by-one for those not in your environment:

  • h5py-2.3.1.win-amd64-py2.7.exe
  • matplotlib-1.4.2.win-amd64-py2.7.exe
  • numpy-MKL-1.8.2.win-amd64-py2.7.exe
  • Pillow-2.6.1.win-amd64-py2.7.exe
  • pyparsing-2.0.3.win-amd64-py2.7.exe
  • python-dateutil-2.2.win-amd64-py2.7.exe
  • six-1.8.0.win-amd64-py2.7.exe
  • wxPython3.0-win64-3.0.2.0-py27.exe
